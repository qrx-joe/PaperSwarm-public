# Education Quasi-Experiment Case Study

This case shows the base four-role review and a real Worker/Verifier revision
loop.

It is the strongest public evidence for the `revise` step: unlike the medical
RCT architecture-validation run, this run includes `worker-round1`,
`verifier-round1`, `worker-round2`, `verifier-round2`, `paper_revised.md`, and
`revise_report.md`.

## What It Demonstrates

- Education quasi-experiment papers can use the base reviewer swarm:
  `editor`, `method`, `domain`, and `devil`.
- The central defect is design-level: whether missing pretest data makes the
  causal claim unrecoverable.
- The Worker/Verifier loop catches superficial or incorrect revisions and
  converges only after independent verification.
- The final archive records the revision loop as reusable experience.

## Public Boundary

runtime state. The included GEP bundle and preview are enough to inspect the
asset-packaging path without exposing credentials.
