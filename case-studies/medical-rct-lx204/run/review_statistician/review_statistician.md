## 具体意见（引用论文具体统计量/段落）

### 问题 1【最严重 · 主终点偏倚】LOCF 在脱落不对称时方向可预测，且未做敏感性分析

**引用**：第 2.4 节「缺失数据以末次观测结转法（LOCF）填补」；第 3.1 节「LX-204 组完成 98 例（脱落 22 例，18.3%），安慰剂组完成 84 例（脱落 36 例，30.0%）」；第 3.2 节「按 ITT 人群（含 LOCF 填补）分析，结果一致」。

**问题剖析**：
- 脱落率**严重不对称**（LX-204 18.3% vs 安慰剂 30.0%）。在 T2DM 这种进行性疾病中，未治疗/安慰剂组病情倾向于随时间恶化（HbA1c 上升），LOCF 把脱落前的较早观测"结转"到第 12 周，会**系统性高估**安慰剂组的终点 HbA1c 改善（即把本应恶化的患者按早期较好看的数据冻结），从而**缩小** LX-204 与安慰剂的差值 → LOCF 在此处**倾向于保守、低估疗效**。
- 但反向看：LX-204 组脱落率更低，意味着更多真实有效数据进入分析，而安慰剂组更多被 LOCF 冻结在较早期数值——如果安慰剂组后期恶化，LOCF 又会让安慰剂组"看起来没那么差"，方向恰好相反于上一条。**两种力量叠加，偏倚方向并非作者一句话"结果一致"可搪塞**，必须用敏感性分析定量。
- **缺失机制（MCAR/MAR/MNAR）只字未提**。脱落不对称本身已强烈提示非 MCAR（脱落可能与疗效/不良事件相关 → MAR 或 MNAR），LOCF 的有效性前提（近似 MCAR）在此处不成立。
- LOCF 已被 ICH E9(R1)、FDA 及 EMA 明确列为**不推荐**的缺失数据处理方法，2020 年后的糖尿病 RCT 主流是 MMRM（混合模型重复测量）+ tipping point / 多重填补敏感性分析。

**正确统计做法**：
1. 主分析改用 **MMRM**（重复测量混合模型，非结构化协方差，含访视×治疗组交互项），对缺失依赖 MAR 假设。
2. 必须报告 **缺失数据机制假设**并讨论合理性（脱落原因是否与 HbA1c 相关）。
3. 强制 **敏感性分析**：(a) 多重填补（MI，含 MNAR 情景如 δ-adjustment / tipping-point analysis）；(b) 最坏病例填补（对照组填最差、试验组保持）；(c) 完成病例分析作为参照。报告 tipping point（δ 多大时主要终点不再显著），这是 FDA 当前明确要求格式。
4. 报告**各组各访视的缺失数量与原因**（CONSORT flow + 缺失原因表）。

### 问题 2【最严重 · 多重比较】8 个次要终点无任何多重比较校正，family-wise error 不可控

**引用**：第 3.3 节「LX-204 组空腹血糖下降 1.8 mmol/L（p=0.01）、体重下降 2.6 kg（p=0.03）、腰围下降 2.1 cm（p=0.04），均显著优于安慰剂组。对 8 个预设次要终点进行比较」。

**问题剖析**：
- 8 个次要终点全部按**名义 p<0.05** 报告，无 Hochberg、Holm、Hochberg-Bauer、固定顺序层级（hierarchical gatekeeping）或贝叶斯方法校正。在 α=0.05、8 个独立检验下，**family-wise type I error 至少 1−(0.95)^8 ≈ 34%**，即三分之一概率至少有一个假阳性。p=0.03、p=0.04 的结果在多重校正后（如 Holm：最小 p 需 ≤ 0.05/8=0.00625；Hochberg 同理）**几乎必然不显著**。
- 摘要与讨论直接将"空腹血糖、体重亦显著改善"作为定性结论，**未区分主要 vs 次要终点的证据强度**，读者会误读为确证性证据。
- 论文声称"8 个预设次要终点"，但**未在方法（2.3/2.4 节）列出这 8 个终点是什么**、也未说明是否事先预设层级或 α 分配方案（alpha-spending）。

**正确统计做法**：
1. 方法节明确列出全部 8 个次要终点，并事先声明**多重比较策略**：推荐 **hierarchical fixed-sequence**（按预先排序，前一个显著才检验下一个，FWER 控制 α）或 **Hochberg procedure**。
2. 摘要、结果、讨论必须报告**校正后 p 值与 95% CI**，并对名义 p 显著但校正后不显著者明确标注"探索性，不构成确证证据"。
3. 区分**确证性终点**（仅 HbA1c 主终点）与**支持性/探索性终点**，后者结论用词应降级（"提示""趋势"而非"显著改善"）。

### 问题 3【数据自洽性】主终点 CI 与 p 值不自洽，疑似选择性报告或计算错误

**引用**：第 3.2 节「组间差值 1.1%（95% CI 0.6–1.6%，p=0.02）」；摘要同样数据。

**问题剖析**：
- 差值 1.1%，CI 宽度 1.0%（0.6–1.6），即 SE ≈ 0.255%（CI 半宽 / 1.96）。
- 在 n≈120/组、SD 约 0.85（ pooled ）下，两组均值差的 SE = √(0.85²/120 + 0.85²/120) ≈ 0.110，对应 z = 1.1/0.110 ≈ 10 → **p 应远小于 0.001**。
- 即便按 ITT 全样本但用更保守的 ANCOVA 残差 SE，z 也应在 6–8 量级，p < 0.001。论文报 p=0.02（对应 z≈2.3）**严重高估** p 值，**与所给 CI 完全不自洽**。
- 自洽性推演：若 p=0.02 属实，则 SE 应 ≈ 0.478，CI 应约为 [0.16, 2.04] 或更宽，与所报 [0.6, 1.6] 矛盾。
- 三种可能：(a) CI 写错；(b) p 值算错/取错（如来自未调整模型或脱落病例子集）；(c) 选择性报告（挑了对"刚好显著"看起来更稳的组合）。无论哪种，**主终点数据不可信**，必须复核原始分析输出。

**正确统计做法**：
1. 重新计算并报告：ANCOVA 估计的组间最小二乘均值差、SE、95% CI、t 与 p 值，附分析软件与版本（如 SAS PROC MIXED / R lme4）。
2. CI 与 p 必须由同一模型同时输出，避免人工拼接。
3. 在补充材料中公开分析代码或至少分析计划（SAP）。

### 问题 4【样本量】240 例无事先功效计算推导，可重复性为零

**引用**：全文仅第 2.1 节「纳入 240 例」、摘要「纳入 240 例」，**无任何样本量计算段落**。

**问题剖析**：
- 没有 α、power（1−β）、假设组间差值 δ、假设 SD、脱落补偿系数的推导。读者无法判断 120/组是否足以检出 1.1% 差值。
- 倒推核验：若设 α=0.05（双侧）、power=80%、δ=1.0%、SD=0.9%（取自结果），单尾/双尾双侧检验每组需 ≈ √2 × (1.96+0.84) × 0.9 / 1.0 ≈ 3.56 → n≈13/组？这显然与 120/组不符，说明真实假设差值更小或 SD 更大；或 120/组是为更小的 δ（如 0.5%）设计。**作者必须公开原始假设**。
- 若是为 δ=0.5%、power=90% 设计，而实测差值 1.1% 远超假设 → 功效过剩，反推 LOCF 保守偏倚可能掩盖了更大真实效应。这些推演没有原始 SAP 都是空谈。

**正确统计做法**：
1. 方法节新增「样本量」段落：明确 α、power、假设 δ（引用既往文献或预试验）、SD、单/双侧、脱落补偿、所用软件（如 PASS / G*Power / R pwr）。
2. 报告**实际达到的 power**（observed power / post-hoc）作为参考，但避免据此改写结论。
3. 若试验有注册（ClinicalTrials.gov / ChiCTR），注册号与 SAP 应在方法节披露。

### 问题 5【ANCOVA 模型】仅基线 HbA1c 单协变量，未纳入随机化分层因素

**引用**：第 2.4 节「主要终点采用 ANCOVA 模型，以基线 HbA1c 为协变量」；第 2.2 节「采用中央随机化系统按 1:1 分组，区组大小为 4」（未提分层）。

**问题剖析**：
- ANCOVA 仅含基线 HbA11c，**未纳入背景治疗（二甲双胍剂量/疗程）、中心（多中心试验必须作为分层或随机效应）、BMI、年龄、性别**等已知 HbA1c 强预测因子。ICH E9 与糖尿病 RCT 惯例要求 ANCOVA 至少纳入**基线 HbA1c + 中心 + 随机化分层因素**。
- 多中心试验不调整中心效应 → 中心间患者分布不均会污染组间比较（center × treatment confounding）。
- 模型未声明**是否含治疗组 × 基线交互项**。若存在交互（基线越高降幅越大，见 3.4 节子组），主效应 ANCOVA 不成立，应报告交互 p 并按基线分层报告效应。

**正确统计做法**：
1. ANCOVA 主模型应含：**基线 HbA1c + 治疗组 + 中心（固定效应或随机效应）+ 随机化分层因素**。
2. 若检验治疗 × 基线交互显著，主终点应改报**协变量调整后的均值差**并在预设基线值（如中位数）处给出估计。
3. 报告模型假设检验（残差正态性、方差齐性），ANCOVA 在 SD 较大或非正态时应改用 robust SE 或 bootstrap。

### 问题 6【子组分析预设性】子组未声明是否事先预设，无交互检验

**引用**：第 2.4 节「子组分析按基线 HbA1c（<8.5% vs ≥8.5%）分层」；第 3.4 节「基线 HbA1c ≥8.5% 组降幅（−1.7%）大于 <8.5% 组（−1.1%）」；第 4 节讨论直接升华「基线 HbA1c 较高者获益更大，与 GLP-1 机制依赖残存 β 细胞功能的假说一致」。

**问题剖析**：
- 子组分析是 RCT 中**最容易产生假阳性**的环节。本文：
  - 未声明该子组切点（8.5%）是否来自 SAP 事先预设（8.5% 在方法节出现过，但未说"事先""预设")；
  - 未报告**交互 p 值**（interaction test），仅比较两组降幅数值；
  - 未对该子组做**多重比较校正**（属额外比较）；
  - 讨论直接把子组结果当确证结论做机制推导 → 严重过度解读。
- −1.7% vs −1.1% 在没有 SE/CI/交互 p 的情况下，差异可能纯属抽样波动。

**正确统计做法**：
1. 方法节明确：所有子组分析是否事先在 SAP 中预设，切点依据（临床/统计）。
2. 子组必须报告**交互 p**（治疗组 × 子组变量），交互 p 显著才支持效应修饰结论。
3. 子组结果用**森林图 + 95% CI** 呈现，标注"探索性"。
4. 讨论对子组结论用词必须降级为"提示性""需进一步验证"。

### 问题 7【效应量与 CI 报告完整性】次要终点与安全性缺 CI，仅报 p 值

**引用**：第 3.3 节次要终点「下降 1.8 mmol/L（p=0.01）」「下降 2.6 kg（p=0.03）」「下降 2.1 cm（p=0.04）」；第 3.5 节安全性「轻度低血糖 6 例 vs 2 例」「胃肠道不良反应 24 例（20%）vs 8 例（6.7%）」。

**问题剖析**：
- 次要终点**无组间差值、无 95% CI、无 SD/SE**，仅报 p——违反 Lancet/NEJM/JAMA 共同的统计分析报告规范，且使读者无法判断效应大小与精确度。p=0.04 的 2.1 cm 腰围差值，若 CI 宽到 [0.1, 4.1]，临床意义存疑。
- 安全性事件无**相对风险（RR）/比值比（OR）及 95% CI**，也无组间比较 p（或恰当的 Fisher exact）。GI 不良反应 20% vs 6.7% 看似明显，但未给 RR（≈3.0）和 CI，无法判断是否排除偶然。

**正确统计做法**：
1. 所有连续型次要终点报告：组间最小二乘均值差 + 95% CI + 校正后 p。
2. 安全性二分类事件报告：RR/OR + 95% CI + 确切检验 p（事件数小时用 Fisher exact 或 mid-p）。
3. 时间-事件型安全性（如至首次低血糖时间）若适用，用 KM + log-rank / Cox HR。

### 问题 8【次要 · 缺失与 ITT 定义】ITT 与 LOCF 混用，定义不清

**引用**：第 2.4 节「采用 ITT 原则……缺失数据以 LOCF 填补」；第 3.1 节未给 ITT 人群与 PP 人群各自的 N。

**问题剖析**：
- ITT 原则要求所有随机化患者纳入分析，LOCF 是实现 ITT 的一种（已过时）方法，但论文未声明：是否包含**随机化但未接受治疗**的患者？是否排除**基线后无任何 post-baseline 数据**者（modified ITT）？
- 未报告**PP（per-protocol）人群**作为敏感性分析。

**正确统计做法**：
1. 明确 ITT 人群定义（全随机化 vs modified ITT），给出各组 N。
2. PP 人群与 ITT 结果并报，差异讨论。

## 修改建议（按优先级排序，作者 to-do）

1. **【必改 · 阻断性】**重做主终点缺失数据处理：LOCF → **MMRM**（MAR 假设）+ **MNAR 敏感性分析**（tipping point / δ-adjustment）；报告各组各访视缺失数与原因。重算主终点 CI 与 p，确保自洽（解决问题 1、3、8）。
2. **【必改 · 阻断性】**补**事先功效计算**段落（α、power、δ、SD、软件）；披露试验注册号与 SAP（解决问题 4）。
3. **【必改 · 阻断性】**8 个次要终点做**多重比较校正**（建议 hierarchical fixed-sequence 或 Hochberg），报告校正后 p 与 95% CI，摘要/讨论对校正后不显著者降级用词（解决问题 2、7）。
4. **【必改】**ANCOVA 模型纳入**中心 + 基线 HbA1c + 随机化分层因素**；检验治疗 × 基线交互，若显著则分层报告（解决问题 5）。
5. **【必改】**子组分析：报告**交互 p**，声明是否事先预设，讨论降级为"探索性"（解决问题 6）。
6. **【建议】**公开分析代码或 SAP；公开个体患者数据（IPD）声明，符合 ICMJE 数据共享要求。
7. **【建议】**所有效应量补 95% CI 与 SD/SE；安全性补 RR/OR + CI + 确切检验 p。

## 结论

本文主要终点（HbA1c −1.1%）的统计证据链存在三重阻断性缺陷：**LOCF 在脱落不对称下偏倚方向不明且未做敏感性分析**、**主终点 CI 与 p 不自洽疑似计算/报告错误**、**8 个次要终点零多重校正**。仅凭现有统计报告，主要与次要终点的结论均不构成可靠证据。建议作者在补做 MMRM + 多重填补敏感性分析、补样本量推导、补多重比较策略后重新提交。
