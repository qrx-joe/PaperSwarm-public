# 评审 Rubric：5 维度质量清单

> 用途：注入评审 subagent 的 prompt，作为评分维度。本清单为 PaperSwarm 独立设计，按论文类型可插拔组队（见 structure 产出的 reviewer_set.json）。

## 5 维度 + 权重

| 维度 | 权重 | 核心检查 |
|------|------|----------|
| 1. 原创与贡献 | 25% | 研究缺口陈述 / 创新点明确性 / 学术与实践重要性 / 与既有理论的对话深度 |
| 2. 方法严谨性 | 25% | 研究设计与问题匹配 / 推理链条自洽 / 方法选择论证充分 |
| 3. 证据与可复现性 | 20% | 数据来源可溯源 / 效应量与统计前提 / 代码与材料开放可复跑 |
| 4. 论证与表达 | 15% | 主张-子主张-证据链对应 / 段落逻辑递进 / 核心结论有数据支撑 |
| 5. 规范与伦理 | 15% | 学术用语与符号规范 / 人体研究伦理审查 / 利益冲突披露 |

## 评分标尺
- 8.5–10：达到投稿水准
- 7.0–8.4：需小修后投稿
- 5.5–6.9：需大修
- <5.5：建议重写

## 角色 → 维度映射（决定每个 subagent 注入哪些维度；角色池见 reviewer_set.json）
- 主编 editor → 1（整体贡献判断）, 4, 5（规范侧）
- 方法论 method → 2, 3（证据侧：设计/数据收集）
- 领域专家 domain → 1（文献前沿/理论对话/缺口成立性）
- 魔鬼代言人 devil → 全维度对抗，重点 3（证据真实性）
- 统计 statistician → 3（统计侧：前提检验/效应量/模型选择）
- 复现 reproducibility → 3（复现侧：代码数据开放/可复跑）
- 伦理 ethicist → 5（伦理侧：IRB/知情同意/利益冲突）

## 常见缺陷速查（9 类）
1. 引言未点明研究缺口，读不出"为何要做这个"
2. 文献综述按时间堆叠，缺主题脉络与定位
3. 方法步骤不可复现，关键参数与流程缺失
4. 结果仅描述无效应量，定量结论无支撑
5. 讨论复述结果，未升华到贡献与局限
6. 结论空泛，与正文证据脱钩
7. 参考文献格式不一，正文引用与文末条目错位
8. 图表无解读，正文未呼应图表信息
9. AI 生成痕迹明显（套话堆叠、对称句式、空洞过渡）
